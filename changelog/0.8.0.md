## [0.8.0] — 2026-04-24 · Browser mode, pair picker, Source Map, page-layout wireframe

The "make it genuinely hostable" release. 0.7.0 polished the MD output; 0.8.0 ships everything around it: a production browser deploy via GitHub Pages, a pair-validation picker that handles multi-project folders, a Source Map tab for data-engineer impact analysis, and a filtered SVG page-layout wireframe. 25 PRs merged.

### Added

**Browser mode (install-free, runs from GitHub Pages):**
- Static deploy workflow (`.github/workflows/pages.yml`). Every push to `main` rebuilds `docs/` and publishes to `https://jonathan-pap.github.io/PowerBI-Lineage/`.
- Browser entry shell (`src/browser/entry.ts` + `fs-shim.ts` + `path-shim.ts`) — the existing parser runs unchanged against an in-memory VFS populated via File System Access API.
- "Try a sample" button with a baked PBIP (`scripts/bake-sample.mjs` + `docs/sample-data.json`). One-click demo for visitors without a project file, and the Firefox/Safari escape hatch.
- "Documenter" landing-page framing — outcome-first subhead, feature grid, trust chips (157 tests · Zero deps · Files stay on your machine), CLI-fallback link for non-Chromium browsers.

**Pair picker (replaces multi-step direct-picks):**
- Parent-folder picker detects every `*.Report` + `*.SemanticModel` pair under the picked root. Single pair + validated pairing → auto-loads (zero extra clicks). Multi-pair or mismatch → shows a radio-group selection UI.
- Three-tier pair validation: (1) `definition.pbir` pointer match, (2) name-prefix heuristic, (3) mismatch → Load disabled with explicit error.
- Symmetric auto-select: clicking Report flips Model to the match, and vice-versa. Pbir-first with prefix-exact preference inside pbir matches for the Model→Report direction.
- Report-only and Model-only load modes — `(none)` radio on either side synthesises an empty shim for the missing half. Header badge (purple "Model only" / green "Report only") surfaces partial-mode state at a glance.
- Wider overlay card for the picker (min(960px, 95vw)) with smooth transition; landing card stays narrow.

**Source Map tab (inside a merged Sources tab):**
- Flat row-per-column view mapping every PBI column to its physical source (type, server/DB, source location). Filterable by source type, searchable, CSV-exportable.
- Sources tab gains a segmented `[📋 Cards] [📊 Flat map]` view toggle — Cards view = existing partition breakdown (architect-oriented), Flat map = Source Map (data-engineer-oriented). Tab bar shrinks by one entry.

**Page-layout SVG wireframe (revived from `parked/wireframe-poc`):**
- "Layout" section at the top of each expanded page card shows a scaled PBI-canvas SVG with coloured rectangles per visual at their true positions. Collapsible via native `<details>` (default closed) because the SVG is vertically significant.
- Category whitelist: only chart / table / card / slicer / map / AI visuals render. Shapes, buttons, text, images, unknowns excluded upstream in `categorizeVisualForWireframe()` — "over kill" avoided per user feedback.
- Size-gated inline labels: visuals under 150×80 render as coloured rectangles with a hover-only tooltip; larger visuals keep their inline type + title. Slicers exempt from the size gate (always labelled).
- Hover tooltip now shows field-well bindings grouped by bucket role (Values, Category, Y, Rows, Tooltips, …). Filter bindings deliberately excluded.

**Docs tab additions:**
- New "Changelog" entry (10th doc). Project-level metadata — not overwritten when the user loads a different report.

**Header:**
- "Load another" button (repurposed from `Re-scan`). Re-opens the overlay without a page refresh, preserving tab state and avoiding a re-fetch of the 2 MB sample-data.json.

### Changed

- **Repo name stays `PowerBI-Lineage`**, but UI framing shifts to "Power BI Documenter" (headline + tagline + browser `<title>`). Lineage is still a feature (Mermaid DAGs in measures.md, star fragments in data-dictionary.md); it's no longer the whole-product story.
- **Dim text tiers brightened** for dark-mode legibility on small (12px) text. `--text-dim` `#7A8595` → `#8796A8`, faint + fainter shifted equivalently. Hierarchy preserved.
- **Top-level script bindings** in the inline HTML `<script>` switched from `const` → `let` (`DATA`, all `MARKDOWN_*`, `REPORT_NAME`, `GENERATED_AT`, `APP_VERSION`). Required for the browser-mode `__loadBrowserData` hook to swap the dashboard's backing data without a page reload.
- **`generateHTML` signature** grew from 12 to 13 arguments (adds `changelogMarkdown`). Defaulted to `""` so existing callers keep working.

### Fixed

- Dashboard rendering an empty shell when the browser-mode sample/folder-pick fired before the script-scope globals could be swapped — root-caused to closure over `const DATA`.
- Error message when a user steps INTO a `.Report` folder during the pick — now a specific self-correcting message explaining the sibling-access constraint.
- Source Map tab bootstrap hang at "Rendering docs…" caused by an undeclared `isAutoName` reference in `sourceMapRows()` — removed.
- `.rb-report` header element updating on load, but the visible `.header-sub` span not. Both (plus `.timestamp` + `document.title`) now refresh on every load.
- `&nbsp;` rendering literally in External-proxy-measures group headers (MD renderer treats HTML entities as plain text inside headings). Replaced with U+00A0.
- README anchor in the browser-mode footer pointing at `#cli-mode` which didn't exist → `#running`.

### Infrastructure

- GitHub Pages deploy workflow (`.github/workflows/pages.yml`) with `enablement: true` for first-run provisioning and `concurrency: pages · cancel-in-progress: true` for rapid-push safety.
- Build pipeline grew to four steps: `tsc` (CLI) → `tsc -p tsconfig.browser.json` (browser) → `scripts/build-browser.mjs` (shell) → `scripts/bake-sample.mjs` (sample payload).
- `docs/sample-data.json` committed as a build artefact (with `.gitignore` negation for `docs/*`); source `sample/` stays local-only.
- Repo metadata: description, homepage URL, 16 topics set on GitHub for discoverability.

### Security

- Sample-data sensitivity scan — zero credential hits, all external URLs are Microsoft Fabric schema refs. Safe to ship publicly.
- XSS fuzz test suite extended to cover the new Source Map tab (all `data-*` attributes go through `escAttr()`).
