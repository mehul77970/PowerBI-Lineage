## [0.10.2] — 2026-04-25 · erDiagram render fix + Measures dep-chip links

Two render-quality bugs caught on the live GitHub view of the generated MDs.

### Fixed

- **erDiagram fell back to plain code on GitHub.** Power BI's `_measures` / `_Rollup_measures` underscore-prefix convention produced entity names starting with `_` in our Mermaid `erDiagram` output. Mermaid's grammar requires entity identifiers to match `[A-Za-z][A-Za-z0-9_]*` — letter first. GitHub silently falls back to rendering the whole code block as plain code when parsing fails.

  Fix: `erIdent()` now prefixes `E` to any name whose first surviving char isn't a letter. `_measures` → `E_measures`, `_Rollup_measures` → `E_Rollup_measures`. Existing all-letter / digit-leading guards preserved.

  Verified: H&S fixture's 30 entities (capped per `MAX_ER_TABLES`) all render cleanly on GitHub now; were previously dropping to code-block fallback.

- **Measures.md "Depends on" / "Used by" chips were styled but unlinked.** F13 in PR #86 added cross-doc links across other docs but missed the in-doc dependency chips on each measure's `<details>` block. Reader saw `Gross Revenue` as styled text instead of clickable link to the dependency's A–Z entry.

  Fix: chips now wrap `xref(name, null, name)` so each chip becomes `<span class="chip chip--measure">[Gross Revenue](#gross-revenue)</span>`. Dashboard MD viewer's anchor-scroll handler picks them up; ADO Wiki + GitHub render the inner anchor as a regular link.

  Verified: H&S fixture emits 96 chip-links across all per-measure entries.
