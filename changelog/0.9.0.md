## [0.9.0] — 2026-04-24 · Data-engineer depth pass

Four substantive additions driven by a side-by-side competitive audit against pbip-documenter. Each gap closes with server-side parser work plus surface in the dashboard and the generated Markdown. 237 tests, bundle 265 KB.

### Added

- **Relationship cardinality + cross-filter direction** (PR #71) — the TMDL / BIM relationship parser was reading but silently dropping `fromCardinality`, `toCardinality`, and `crossFilteringBehavior`. Now surfaced in three places:
  - **Relationships tab** — new Cardinality (`* → 1`) and Filter (`↔ both` / `→ single`) columns
  - **Tables tab per-rel chip** — compact `*:1` label plus a `↔` icon when bidirectional
  - **Model.md §2.3** — inventory table gains Cardinality and Filter columns

  Defaults per TMDL spec (`many → one`, `oneDirection`) are materialised explicitly so downstream code never has to second-guess an empty string.

- **`Value.NativeQuery` SQL extraction** (PR #71) — new `extractNativeQuery()` walks an M expression with paren-depth + string-literal tracking to pull the hand-written SQL out of `Value.NativeQuery(<conn>, "<SQL>", …)` calls. Falls back to the `Sql.Database(…, [Query="…"])` inline options-record form. Surfaced as:
  - **Sources tab "Native queries" card** — monospace SQL blocks per partition
  - **Sources.md §2.1** — fenced ` ```sql ` blocks per partition

  Handles multi-line SQL, M's `""` embedded-quote escape, nested function calls in the connection argument. Guards against false-positive `Query="some.csv"` attributes by requiring `SELECT` / `WITH` / `EXEC` in the match.

- **Broken-reference detection** (PR #72, hotfixed in PR #73) — new Improvements check scans every DAX-carrying expression (measures, calc-group items, UDF bodies) and flags references to tables / columns / measures that don't exist in the model. Common cause: a table or column was renamed / deleted and callers were missed.

  Two new public exports in `src/improvements.ts`:
  - `extractDaxRefs(dax)` — pulls qualified `Table[Col]` and bare `[Name]` refs out of DAX after stripping comments + string literals
  - `brokenReferences(data)` — returns `BrokenRefFinding[]` with location + broken ref + reason

  Resolution rules mirror DAX's permissive semantics: `Table[X]` resolves if X is either a column on Table or a measure anywhere (DAX accepts `Table[Measure]`); bare `[X]` resolves against any measure or column (row context makes unqualified refs legitimate); `EXTERNALMEASURE` proxies are skipped. Case-insensitive matching (fixed in PR #73) — DAX identifiers are case-insensitive at query time.

  Smoke-tested on the H&S fixture: zero false positives after the case-insensitive hotfix.

- **M-step parser** (PR #74) — new `parseMSteps(m)` hand-rolled walker classifies every step in a partition's `let … in` body into ten kinds: `source` · `navigation` · `projection` · `rename` · `filter` · `join` · `addColumn` · `typeChange` · `expand` · `custom`. Mirrors pbip-documenter's taxonomy.

  Surfaced as:
  - **Sources tab "M-step breakdown" card** — per-step rows with colour-coded kind chips, function name, and best-effort detail summary (filter column, join tables, added column name, …)
  - **Sources.md §2.2** — table with `#`, Step, Kind, Function, Detail columns

  Parser handles string literals (with M's `""` escape), `/* */` + `//` comments, nested brackets, `each let … in …` lambdas (must not promote their steps to the outer level), quoted `#"Step Name"` identifiers, three-part namespaced custom connectors (`MyCompany.Custom.ScalarFn`), and ~30 connector clusters (`Sql`, `Web`, `Csv`, `Excel`, `Json`, `SharePoint`, `BigQuery`, `Databricks`, `AnalysisServices`, …). Power BI's "Enter Data" shape (`Table.FromRows(Json.Document(Binary.FromText(…)))`) classifies as source with `Table.FromRows` as the primary function rather than the decode-layer inner call.

  Never throws — unparseable M returns `[]` and the card is hidden.

- **`RawPartition.nativeQuery` + `.steps`** — new persistent fields on every partition record, populated by both the TMDL and BIM parser paths.

### Changed

- **`ModelRelationship` interface** — now includes `fromCardinality`, `toCardinality`, `crossFilteringBehavior`. Callers constructing synthetic relationships (test fixtures) need the three new fields.
- **`TableRelationshipRef` interface** (dashboard data) — same three fields threaded through for per-table display.
- **`runImprovementChecks()`** — adds the broken-reference check between the circular-measure check and the unused-measures check.
- **About popup** (WHATS-NEW.md) — now advertises the new Sources-tab depth and the 16-check audit.
- **Launcher** (PR #70) — `launch.bat` now fast-forwards `origin/main` on each run when the working tree is clean, and rebuilds when HEAD moves or `dist/` is missing. Stale clones auto-upgrade to the latest build on next double-click. Skips the pull when there are uncommitted changes (protects dev work) or `.git` is absent (zip download).
- **CLI documentation** (PR #70) — WHATS-NEW.md gains a "Running locally (CLI mode)" section so Firefox / Safari users reading the About popup know exactly how to use the tool without FSA.
