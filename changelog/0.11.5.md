## [0.11.5] — 2026-04-26 · Improvements: name the entities missing descriptions

The three "X% of tables / measures / columns lack descriptions" checks already existed but only reported the count and percentage — there was no way to act on them without leaving the dashboard. Now each check lists the actual offenders inline, capped like every other items-bearing check.

### Changed
- **Improvements → tables-lack-descriptions** — adds an inline list of the table names missing descriptions, sorted, capped at 15.
- **Improvements → measures-lack-descriptions** — adds an inline list of `Table[Measure]` names, sorted, capped at 15.
- **Improvements → columns-lack-descriptions** — adds an inline list of `Table[Column]` names, sorted, capped at 20.

### Notes
- Severity tiers and percentage thresholds are unchanged (≥30% for tables/measures = medium, ≥50% for columns = low). The thresholds gate noise; the lists make the gated findings actionable.
- Total improvement-check count stays at 16 — these are existing checks enriched, not new ones.
- No version skip — 0.11.4 was used briefly for the GoatCounter visitor counter and reverted in [#99](https://github.com/jonathan-pap/PowerBI-Lineage/pull/99). Going to 0.11.5 keeps version numbers monotonic for anyone who pulled main between the two merges.
