## [0.11.2] — 2026-04-25 · Lineage typeahead — fix empty-state bleed-through

The 0.11.1 lineage search bar shipped with two visible issues on themes where `--surface` is translucent (BluPulse most notably):

- The dropdown used `background:var(--surface)` which is rgba(255,255,255,0.04) under BluPulse, so the lineage-content empty-state prompt bled through behind the typeahead rows.
- The empty-state placeholder ("Type above to search…") stayed visible while the user was typing, competing with the dropdown for attention.

### Fixed

- **Dropdown background switched to `var(--tooltip-bg, var(--bg))`** — solid in every theme. Same approach we already use for stat-card tooltips. Added a subtle box-shadow + bumped z-index to 50 so the dropdown reads as a floating popover.
- **Empty-state hides while typing.** The lineage-content placeholder is cleared the moment the search query has text, restored when the query clears OR when the user dismisses the dropdown via outside-click without selecting anything. Real lineage views (presence of `.lineage-hero`) are preserved across all transitions.

- `npm test` — 267/267 (no test count change — visual fix)
- `npm run build:browser` — 326 KB (under 400 KB cap)
- Manual: BluPulse theme → Lineage tab → type "d" → typeahead is opaque, empty-state is gone. Clear input → empty-state returns. Click outside without selecting → empty-state returns.
