## [0.11.3] — 2026-04-26 · Mobile responsive — sample loads cleanly on a phone

The dashboard was desktop-first and the live demo at jonathan-pap.github.io/PowerBI-Lineage was rough on mobile — tabs overflowed off-screen, the stat-cards row crammed, the landing card overpadded itself off the viewport, modals were padding-trapped. This release fixes the critical surfaces so a LinkedIn click-through from a phone lands somewhere usable.

### Added

- **`@media (max-width: 720px)` block** — tablet portrait + phone landscape. Tabs gain horizontal scroll (with a subtle scrollbar). Header wraps. Search rows give the input full width and let buttons wrap below. Stat cards tighten. Table cells shrink one font-size step. Lineage hero meta condenses.
- **`@media (max-width: 480px)` block** — phone portrait. Stat cards drop to 2 per row. Tab counts hide (badge noise at this size). Container padding tightens further. About / What's-new modal goes near-fullscreen with edge gutters instead of being padding-trapped.
- **Landing card mobile rules** in `build-browser.mjs` — at ≤480px the card pads down, the heading drops to 22px, trust line wraps, CTAs stack as needed.

### What's intentionally NOT mobile-optimised

- **Pages tab wireframe** — true-position SVG of 30+ visuals doesn't rescale meaningfully below ~600px wide. Degrading it would hurt the desktop signal more than helping mobile. Wireframe scrolls horizontally inside its container.
- **Wide tables** (Measures / Columns / Source Map) — kept their existing `.table-wrap` horizontal-scroll wrapper. Reflowing dense data into narrow stacked cards lies about the relationships between columns.
- **Lineage canvas internals** — the upstream / downstream column layout already stacks vertically at ≤768px (existing rule). Internal flow stays as-is.

### Approach notes

Mobile audit focused on landing → tabs → primary tabs (Measures / Columns / Sources / Improvements / Docs) — the path a LinkedIn-arriving visitor most likely takes from "what is this?" to "I get it." Surfaces that exist for serious model exploration (Tables drill-down, full Pages wireframes, Lineage internals) deliberately keep their desktop-class behaviour because mobile isn't a serious-exploration form factor for this tool.
